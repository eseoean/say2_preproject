01

프로젝트 개요

본 프로젝트의 목표는 유방암(BRCA)을 대상으로 약물 재창출(Drug Repurposing) 파이프라인을 재현하고, 다중 바이오데이터와 머신러닝 기반 예측을 이용해 우선 검토할 약물 후보를 도출하는 데 있다. 팀원이 구축한 프로토콜 저장소 20260408_pre_project_biso_myprotocol을 기준으로 동일한 방법론을 재현하고, 우리 팀 환경에서 전체 실행 가능성을 확인했다.

분석에는 GDSC, DepMap, LINCS, ChEMBL, DrugBank, METABRIC, Open Targets, STRING, MSigDB, ADMET 등 다양한 공공 바이오데이터가 활용되었다. 이 데이터들은 약물 반응, 세포주 특성, 타깃 유전자, 약물 구조, 외부 검증, 안전성 평가까지 연결되어 하나의 일관된 분석 흐름을 구성한다.

프로젝트 목표

  • 팀원 프로토콜을 기준으로 동일한 파이프라인을 우리 팀 환경에서 재현한다.
  • 다중 데이터 통합 기반 feature engineering과 모델 비교를 수행한다.
  • 앙상블, 외부 검증, ADMET 필터링을 통해 후보 약물을 좁힌다.
  • 최종적으로 대시보드와 보고서 형태로 결과를 정리한다.
02

프로젝트 팀 구성 및 역할

우리 팀은 총 4명으로 구성되어 있으며, 현재 보고서는 역할 중심으로 정리했다. 최종 제출 시에는 실제 이름과 세부 기여도를 반영해 수정할 수 있다.

역할 구분주요 담당 내용대표 산출물
역할 1. 데이터 조사 및 정합성 관리데이터 출처 확인, curated data 구조 이해, provenance 문서 검토, 입력 경로 점검DATA_PROVENANCE.md, dataset inventory 정리
역할 2. 입력 데이터 준비 및 실행 환경 구축AWS/S3/Nextflow 환경 구성, Step 1~3 실행, 재현용 설정 파일 구성project_env.sh, run_fe_pipeline.sh, FE 산출물
역할 3. 모델링 및 검증 파이프라인 운영Step 4~7 실행, 모델 비교, 앙상블, METABRIC 검증, ADMET gate, 결과 해석ml_results.json, ensemble_results.json, step6_metabric_results.json, step7_admet_results.json
역할 4. 프론트엔드 및 결과 전달 레이어대시보드/리포트 화면, 시각화, 사용자 전달 구조 정리dashboard.html, 프론트엔드 화면, 발표 자료
현재 기준으로 파이프라인 재현과 보고서 정리는 완료되었고, 프론트엔드는 팀원이 계속 작업 중이다. 따라서 본 문서는 최종 UI가 완성되기 전 단계의 중간-최종형 보고서 초안으로 볼 수 있다.
03

프로젝트 수행 절차 및 방법

3.1 전체 수행 절차

  1. 환경 설정과 실행 경로 구성
  2. curated data 기반 입력 데이터 준비
  3. Nextflow 기반 Feature Engineering 수행
  4. ML, DL, Graph 모델 학습 및 성능 비교
  5. 앙상블을 통한 예측 안정성 확보
  6. METABRIC 외부 검증 수행
  7. ADMET 기반 최종 후보 약물 선별

3.2 데이터 준비 원칙

  • curated_date/는 읽기 전용으로 사용했다.
  • curated_date/glue/ 경로는 접근하지 않았다.
  • GDSC는 GDSC2만 사용해 품질 우선 원칙을 적용했다.
  • LINCS는 빠른 재현 경로로 reference의 lincs_mcf7.parquet를 사용했다.
  • 중복 약물은 최종적으로 drug_name 기준으로 제거했다.

3.4 Feature Engineering 방법

Step 3은 Nextflow + AWS Batch 기반으로 실행했다. pair feature는 CRISPR 기반 세포주 특성, 약물 구조 정보, LINCS signature, drug-target 정보를 통합하는 구조로 생성했다.

실행 방식

AWS Batch + Nextflow

실행 시간: 약 8분 7초

주요 산출물

features.parquet, labels.parquet, pair_features_newfe.parquet

05

자체 평가 의견

5.1 잘된 점

  • Step 1~7 전체 파이프라인을 실제 환경에서 끝까지 재현했다.
  • AWS Batch, Nextflow, ML/DL/Graph, METABRIC, ADMET까지 전체 흐름을 경험적으로 확인했다.
  • 6-model과 경량형 앙상블을 모두 비교한 뒤, 성능과 운영 효율을 함께 고려해 합리적인 선택을 했다.
  • 대시보드와 보고서까지 연결해 결과 전달 구조를 미리 확보했다.

5.2 아쉬운 점

  • 프론트엔드가 아직 진행 중이어서 최종 화면 캡처를 완전히 반영하지 못했다.
  • 팀원별 실명과 세부 역할 비중은 최종 제출 전에 다시 보정해야 한다.
  • Graph 계열 모델은 이번 재현에서 회귀 예측 성능이 충분히 높지 않았다.

5.3 향후 보완 방향

  • 프론트엔드가 완료되면 현재 대시보드와 통합해 최종 발표 화면을 정리한다.
  • 실제 팀원 이름과 담당 범위를 반영해 02장을 마무리한다.
  • 보고서 최종본에는 대시보드 캡처, 비교표, 후보 약물 카드 이미지를 추가한다.
  • 발표용 버전은 Step 5 경량형 선택 이유와 Step 6, Step 7 검증 근거를 중심으로 더 압축해 정리한다.

5.4 종합 의견

이번 프로젝트는 팀원 프로토콜을 기반으로 약물 재창출 파이프라인을 실제로 재현하고, 외부 검증과 ADMET 필터링을 거쳐 우선 검토할 후보 약물을 도출했다는 점에서 프리프로젝트 목표를 충실히 달성했다고 판단한다. 특히 재현성과 결과 전달성을 동시에 고려했다는 점에서 다음 단계인 프론트엔드 통합과 발표 자료 제작으로 자연스럽게 이어질 수 있다.
